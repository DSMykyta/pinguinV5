# Огляд проєкту — pinguin-v5

> Пояснення структури для людини без технічного фону.

---

## Що це за проєкт?

Це **веб-інструмент для контент-менеджерів** (схоже на внутрішній корпоративний сайт).
Він має кілька окремих сторінок-інструментів, кожна зі своєю задачею.

---

## Сторінки (те, що відкривається в браузері)

| Файл | Що це |
|------|-------|
| `index.html` | Головна сторінка / набір генераторів |
| `mapper.html` | **Маппер** — прив'язка категорій/характеристик до маркетплейсів |
| `brands.html` | **Бренди** — база даних брендів |
| `keywords.html` | **Ключові слова** — бібліотека ключовиків |
| `glossary.html` | **Глосарій** — словник термінів |
| `price.html` | **Прайс** — робота з цінами |
| `tasks.html` | **Завдання** — таск-менеджер |
| `banned-words.html` | **Стоп-слова** — список заборонених слів |

---

## Папка `js/` — серце програми

Кожна папка всередині відповідає одній сторінці або спільному інструменту:

```
js/
├── main-*.js          ← "Вхідна точка" кожної сторінки (запускає все)
│
├── mapper/            ← Весь код сторінки Маппер
├── brands/            ← Весь код сторінки Бренди
├── keywords/          ← Ключові слова
├── glossary/          ← Глосарій
├── price/             ← Прайс
├── tasks/             ← Завдання
├── banned-words/      ← Стоп-слова
│
├── common/            ← СПІЛЬНІ компоненти (використовуються скрізь)
├── generators/        ← Генератори (SEO, таблиці, посилання, зображення...)
├── config/            ← Налаштування (ID таблиць Google Sheets)
├── utils/             ← Маленькі допоміжні функції
├── aside/             ← Бічна панель (права панель з кнопками)
├── auth/              ← Авторизація Google
└── panel/             ← Навігаційна панель
```

---

## Як влаштований типовий модуль (наприклад, Mapper)

Кожна велика сторінка розбита на файли за відповідальністю:

| Суфікс файлу | Що він робить |
|---|---|
| `-main.js` | Запускає модуль, підключає плагіни |
| `-init.js` | Ініціалізація: таби, кнопки, завантаження даних |
| `-data.js` | Завантаження/збереження даних з Google Sheets |
| `-state.js` | "Пам'ять" модуля (що зараз активне, що завантажено) |
| `-table.js` | Відображення таблиці на екрані |
| `-events.js` | Реакція на дії користувача (кліки, введення тексту) |
| `-ui.js` | Відображення елементів інтерфейсу |
| `-crud.js` | Додати / змінити / видалити запис |
| `-polling.js` | Автоматичне оновлення даних у фоні |

---

## Папка `common/` — спільні компоненти

Це речі, які використовуються **на всіх сторінках**:

| Папка/файл | Що це |
|---|---|
| `table/` | Універсальна таблиця з пошуком, сортуванням, пагінацією |
| `editor/` | Текстовий редактор (схожий на Word, але в браузері) |
| `avatar/` | Аватарки користувачів і "порожні стани" (коли нема даних) |
| `charms/` | Маленькі "прибудови" до таблиці: пошук, фільтри, колонки |
| `ui-modal.js` | Спливаючі вікна (попапи) |
| `ui-toast.js` | Маленькі повідомлення ("Збережено ✓") |
| `ui-dropdown.js` | Випадаючі списки |
| `polling.js` | Логіка автооновлення (базова, спільна) |

## Папка `config/` — налаштування

| Файл | Що це |
|---|---|
| `spreadsheet-config.js` | ID Google Таблиць (адреси документів) |

---

## Папка `api/` — серверна частина

Це **бекенд** — код, який працює не в браузері, а на сервері (Vercel). Він потрібен для:
- Авторизації через Google
- Читання/запису Google Sheets
- Роботи з Google Drive
