CSS Design System

Complete documentation of the pinguin-v5 CSS design system. Architecture, tokens, components, conventions, and live examples. 75 CSS files, 4 layers, one consistent approach.

Overview

pinguin-v5 uses a modular, file-based CSS architecture organized into 4 layers: Foundation, Layout, Components, Utilities. All values are centralized as Design Tokens in root.css.

Principles

Core design system principles

#PrincipleDescription
1One File = One ComponentEach CSS file is responsible for exactly one component or functional area
2Cascading Import OrderRoot -> Foundation -> Layout -> Components -> Utilities
3Tokens as Single Source of TruthAll values (colors, spacing, shadows) defined only in root.css
4Dark Mode via Token InversionComponents never have dark-specific styles; theme switch overrides root variables
5No Page-Specific CSSComponents work in any context, never tied to a specific HTML page
6Documentation in Every FileEvery CSS file starts with a JSDoc header block

Architecture

The CSS is organized as an inverted triangle: from the most generic (variables, reset) to the most specific (utilities). All imports flow through main.css.

Theming

Dark mode is implemented by overriding CSS variables in [data-theme="dark"]. Components automatically adapt without any component-level dark styles.

How It Works

/* root.css - Light theme (default) */ :root { --color-surface: #fafafa; --color-on-surface: #171717; --color-main: rgb(9, 63, 69); } /* root.css - Dark theme override */ [data-theme="dark"] { --color-surface: #121212; --color-on-surface: #e5e5e5; --color-main: rgb(76, 166, 175); } /* Components reference tokens only */ .card { background: var(--color-surface); /* auto light/dark */ color: var(--color-on-surface); /* auto light/dark */ border: 1px solid var(--color-outline); /* auto light/dark */ } /* Toggle theme via JS */ document.documentElement.dataset.theme = 'dark'; document.documentElement.dataset.theme = 'light';
RuleDescription
Never hardcode colorsAlways use var(--color-*) — never #hex or rgb() in components
No component dark stylesNever add [data-theme="dark"] .component {} outside root.css
Test both themesEvery component must look correct in both light and dark
Use semantic tokensUse --color-surface not --color-surface-c-low for backgrounds

Accessibility

Guidelines for making the design system accessible. WCAG 2.2 AA compliance as baseline.

Key Requirements

AreaRequirementImplementation
FocusAll interactive elements must have visible focus:focus-visible { outline: 2px solid var(--color-main); }
ContrastText: 4.5:1, Large text: 3:1, UI: 3:1Verify with browser DevTools
MotionRespect reduced-motion preference@media (prefers-reduced-motion: reduce)
TouchMinimum 44x44px touch targetsButtons and links meet this by default
ColorNever convey info by color aloneUse icons + text alongside color indicators
/* Focus visible for keyboard users only */ :focus-visible { outline: 2px solid var(--color-main); outline-offset: 2px; } /* Remove outline for mouse users */ :focus:not(:focus-visible) { outline: none; } /* Respect reduced motion preference */ @media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation-duration: 0.01ms !important; animation-iteration-count: 1 !important; transition-duration: 0.01ms !important; } }
