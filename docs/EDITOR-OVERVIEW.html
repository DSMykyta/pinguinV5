Принципи Editor

ЛЕГО-система для WYSIWYG редактора. Один виклик createHighlightEditor(container) — решта через HTML-чарми.

Живий редактор — всі чарми

Всі доступні функції увімкнено: toolbar, Text/Code, валідація, статистика, тогли очистки, find & replace, undo/redo.

Демо

Спробуйте bold (Ctrl+B), italic (Ctrl+I), заголовки, зміну регістру, пошук, undo (Ctrl+Z). Вставте текст із Word або браузера — редактор автоматично очистить зайве.

HTML — всі чарми увімкнено
<div editor tools code check stats cleanup-links cleanup-styles cleanup-images
     data-editor-id="my-editor"
     data-placeholder="Введіть текст..."
     data-min-height="200"></div>
JS — ініціалізація
import { createHighlightEditor } from './components/editor/editor-main.js';

const container = document.querySelector('[data-editor-id="my-editor"]');
const editor = createHighlightEditor(container);

// Публічне API
editor.getValue();        // → чистий HTML рядок
editor.setValue('<p>текст</p>');
editor.getPlainText();    // → plain text без тегів
editor.clear();
editor.focus();

Чарми — HTML-атрибути

Конфігурація через атрибути на контейнері. Без чарму tools — readonly блок. Кожен чарм незалежний.

Атрибут Що вмикає Залежності
editor Базовий маркер — обов'язковий для ініціалізації —
tools Toolbar + editing + find/replace —
code Перемикач Text / Code (HTML textarea) —
check Валідація: заборонені слова + HTML патерни без tools — readonly + чіпи
stats Футер зі статистикою (символи / слова / хв) —
cleanup-links Тогл: видаляти посилання при копіюванні показує кнопку в футері
cleanup-styles Тогл: видаляти inline-стилі показує кнопку в футері
cleanup-images Тогл: видаляти зображення показує кнопку в футері
data-атрибути
data-editor-id="ghl"      — кастомний ID prefix (default: auto)
data-placeholder="..."    — текст коли редактор порожній
data-min-height="200"     — мінімальна висота в px

Хуки — комунікація між плагінами

Плагіни не знають одне про одного. Вся комунікація — через state.registerHook і state.runHook.

Хук Коли спрацьовує Хто слухає
onInput Будь-яка зміна тексту (input / keyup) undo (debouce), stats, validation (debounce)
onKeydown Кожне натискання клавіші formatting (Ctrl+B/I), undo (Ctrl+Z/Y), enter, paste (Ctrl+Alt+C)
onSelectionChange Mouseup у редакторі formatting (оновлення стану кнопок toolbar)
onWillChange Безпосередньо перед зміною контенту undo (зберегти стан перед змінами)
onValidate Після будь-якої зміни що впливає на контент validation, stats, undo (після undo/redo)
onModeChange Перемикання text ↔ code validation, stats, cleanup
Реєстрація хука в плагіні
export function init(state) {
    // Обов'язково { plugin: 'name' } — для зрозумілих помилок
    state.registerHook('onInput', () => updateStats(), { plugin: 'stats' });
    state.registerHook('onWillChange', saveState, { plugin: 'undo' });

    // Викликати хук
    state.runHook('onWillChange');
    state.runHook('onValidate');
}

Публічне API

Повертається з createHighlightEditor(container). Всі зовнішні взаємодії — тільки через API.

Метод Що робить
getValue() Повертає чистий HTML рядок (без підсвічувань, з урахуванням тоглів очистки)
setValue(html) Встановлює HTML контент і запускає валідацію
getPlainText() Повертає plain text без тегів
getMode() Повертає поточний режим: 'text' або 'code'
setMode(mode) Перемикає режим програмно
focus() Фокус на активному редакторі (text або code)
clear() Очищає контент і запускає валідацію
destroy() Видаляє HTML з контейнера
getState() Повертає внутрішній state (для плагінів / налагодження)
Приклад використання
const editor = createHighlightEditor(container);

// Читати значення при сабміті форми
form.addEventListener('submit', () => {
    const html = editor.getValue();  // чистий HTML
    const text = editor.getPlainText();
});

// Заповнити з бекенду
editor.setValue(dataFromServer.description);

// Очистити після сабміту
editor.clear();
editor.focus();

Написати свій плагін

Плагін — один файл з export function init(state). Додається в масив PLUGINS в editor-main.js.

Шаблон плагіна

Плагін отримує state і може: реєструвати хуки, читати DOM, додавати поведінку. Ніяких прямих імпортів між плагінами.

editor-plugin-my-feature.js
/**
 * ╔══════════════════════════════════════════════════════════════════════════╗
 * ║  🔌 ПЛАГІН — Моя функція                                                  ║
 * ╠══════════════════════════════════════════════════════════════════════════╣
 * ║                                                                          ║
 * ║  Опис що робить плагін.                                                  ║
 * ║  Можна видалити — редактор працюватиме без цієї функції.                ║
 * ║                                                                          ║
 * ╚══════════════════════════════════════════════════════════════════════════╝
 */

export function init(state) {
    // Перевірити чи потрібен плагін
    if (!state.config.myFeature) return;

    const { dom } = state;

    // Підписатись на хуки
    state.registerHook('onInput', () => {
        // реакція на зміни тексту
    }, { plugin: 'my-feature' });

    state.registerHook('onValidate', () => {
        // реакція після валідації
    }, { plugin: 'my-feature' });

    // Перед зміною — зберегти стан для undo
    state.runHook('onWillChange');
    dom.editor.innerHTML = newContent;
}
editor-main.js — додати в масив PLUGINS
const PLUGINS = [
    './editor-plugin-formatting.js',
    // ...
    './editor-plugin-my-feature.js',  // ← додати сюди
];

Конфігурації — наочні приклади

Три типових варіанти: тільки перегляд з перевіркою, редагування з перевіркою, редагування зі статистикою.

editor check — readonly + валідація

Без tools — блок тільки для читання. Заборонені слова підсвічуються, чіпи показують що знайдено. Використовується в модалях перегляду.

HTML
<div editor check data-editor-id="cfg-check"></div>

editor tools check — редагування + валідація

Повноцінний редактор з перевіркою заборонених слів і HTML патернів. Основний варіант для SEO-описів.

HTML
<div editor tools check data-editor-id="cfg-edit"></div>

editor tools stats — редагування + статистика

Без валідації — тільки підрахунок символів, слів та часу читання. Для полів де заборонені слова не потрібні.

HTML
<div editor tools stats data-editor-id="cfg-stats"></div>

editor — тільки виведення тексту

Без жодних чармів — просто readonly блок. Ні редагування, ні перевірки, ні статистики. Для відображення готового тексту.

HTML
<div editor data-editor-id="cfg-view"></div>
