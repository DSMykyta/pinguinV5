Python
Де працює: На сервері або локальному комп'ютері (те, що приховано від користувача).

Основне призначення: Обробка баз даних, штучний інтелект, аналітика, автоматизація процесів та створення серверної частини веб-сайтів (бекенд).

Особливість: Має мінімалістичний синтаксис, структура якого візуально нагадує звичайну англійську мову.

JavaScript (JS)
Де працює: Безпосередньо у веб-браузері користувача (Chrome, Safari тощо).

Основне призначення: Створення інтерактивних елементів на веб-сторінках. Усе, що рухається, оновлюється без перезавантаження сторінки, реагує на кліки (анімації, форми, спливаючі вікна) — працює на JS. Це мова візуальних інтерфейсів (фронтенд).

Особливість: Вбудований у кожен сучасний браузер. Без JS більшість сучасних сайтів будуть виглядати як статичний документ.

Так, сам по собі JavaScript (як рушій V8 у браузері чи Node.js на сервері) легко "вивозить" обробку масивів на сотні тисяч елементів у пам'яті. Мова тут ні до чого.

Причина довгого завантаження ховається в архітектурі твого проєкту. Основні вузькі місця, через які твій мапер на 16 000 категорій гальмує:

1. Google Sheets замість бази даних (Головна проблема)
Google Sheets API працює дуже повільно. Це інструмент для таблиць, а не база даних.

Коли твій бекенд (api/sheets/index.js) робить запит до Google Sheets на витягування 16 000 рядків із характеристиками, сам Google віддає цю інформацію секундами.

Крім того, Google має жорсткі ліміти на кількість запитів (Rate Limits).

2. Розмір переданих даних (Мережа)
Якщо ти тягнеш усі 16 000 категорій з їхніми опціями одним запитом, ти генеруєш величезний JSON-файл (мегабайти даних). Браузеру потрібен час, щоб завантажити цей файл через мережу, а потім розпарсити його на клієнті.

3. Рендеринг DOM (Браузер)
Якщо після завантаження даних ти намагаєшся відмалювати всі ці тисячі рядків у таблиці (mapper.html), браузер просто "ляже". Створення тисяч HTML-вузлів (DOM-елементів) одночасно — це найважча операція для будь-якого браузера.

Як це вирішити (що треба змінити в коді):
На рівні бекенду (API):

Кешування: Не читай Google Sheets при кожному завантаженні сторінки. Налаштуй бекенд так, щоб він раз на день (або по кнопці) витягував дані з таблиці і зберігав їх у швидкий кеш (наприклад, Redis) або просто у статичний JSON-файл на сервері. Клієнт має тягнути дані з цього швидкого кешу, а не чекати відповіді від Google.

Пагінація на сервері: Не віддавай 16 000 записів за раз. Віддавай по 50-100 штук (limit/offset), а решту довантажуй, коли користувач гортає сторінку або переходить на наступну.

На рівні фронтенду (JS/UI):

Ліниве завантаження (Lazy Loading): Завантажуй лише базовий список категорій. Характеристики та опції для конкретної категорії запитуй у сервера (або шукай у масиві) тільки в момент, коли користувач клікає на цю категорію, щоб її розгорнути.

Віртуальний скролінг (Virtual/Infinite Scrolling): Якщо тобі конче треба показати довгий список, використовуй віртуальний скролінг. JS має малювати в HTML лише ті 20-30 рядків, які зараз фізично видно на екрані монітора, підміняючи їх при прокрутці.

JS з цим впорається без проблем, якщо перестати використовувати Google Sheets як пряму базу для "гарячих" запитів і оптимізувати відмальовування в браузері.

Повідомлення Gemini
Твої поточні скрипти виконують інші задачі і не вирішують проблему переповнення DOM-дерева:

lazy-load.js у більшості випадків використовується для відкладеного завантаження картинок або iframe (через IntersectionObserver), коли вони потрапляють у зону видимості екрана. Це не оптимізує рендеринг тисяч текстових рядків бази даних.

polling.js — це механізм регулярних (циклічних) запитів на сервер для оновлення даних. У випадку з важкими запитами до Google Sheets це робить тільки гірше: ти змушуєш браузер і бекенд постійно ганяти туди-сюди важкі масиви даних, забиваючи канал і процесор.

html-loader.js просто підвантажує шматки HTML-коду (шаблони). Це архітектурний інструмент, а не інструмент оптимізації продуктивності рендерингу.

Щоб браузер миттєво малював 16 000 категорій, потрібні специфічні патерни управління даними та DOM-вузлами.

1. Віртуальний скролінг (Virtual Scrolling / DOM Recycling)
Браузер "вмирає" не від розміру JSON-масиву на 16 000 елементів у пам'яті (JS це обробляє за мілісекунди), а від спроби створити 16 000 <tr> або <div> в HTML. Віртуальний скролінг обманює браузер.

Як це технічно працює:

Ти завантажуєш весь масив з 16 000 категорій у змінну JS (наприклад, const allCategories).

Ти вираховуєш загальну висоту списку: 16 000 * 40px (висота одного рядка) = 640 000px.

Ти створюєш порожній контейнер (wrapper) з цією висотою 640 000px, щоб у браузері з'явився правильний повзунок прокрутки.

Всередині контейнера JS фізично створює (рендерить в HTML) лише 30-40 рядків — рівно стільки, скільки влазить на екран користувача + невеликий запас зверху і знизу.

Коли користувач крутить колесо миші (подія scroll), JS не створює нові HTML-елементи і не видаляє старі. Він бере існуючі 40 DOM-вузлів і просто замінює в них текст даними з масиву allCategories, які відповідають поточній позиції скролу. Одночасно контейнер з цими 40 рядками зміщується вниз через CSS transform: translateY(...).

Рішення для твого коду:
Писати віртуальний скролінг з нуля на чистому JS складно через нюанси з подіями миші та розрахунком висоти. Для цього в екосистемі JS є перевірені мікро-бібліотеки, наприклад Clusterize.js. Вона важить 2Кб і створена саме для таких задач на чистому JS (без фреймворків). Вона бере твій масив рядків і сама крутить цей механізм підміни.

2. Ліниве завантаження даних (On-Demand Fetching)
16 000 категорій — це лише назви. Кожна категорія має опції та характеристики. Якщо тягнути їх усі одразу, розмір файлу може складати десятки мегабайтів.

Як це технічно працює:

Первинний запит на бекенд витягує лише ідентифікатори та назви категорій.
[{ id: "cat_1", name: "Вітаміни" }, { id: "cat_2", name: "Мінерали" }, ...]

Цей легкий масив віддається у віртуальний скролер. Користувач миттєво бачить список і може його гортати або шукати по ньому.

Біля кожної категорії є кнопка або іконка "+" (розгорнути деталі).

На цю кнопку вішається обробник події click.

Тільки в момент кліку JS робить запит до бекенду (твого API): fetch('/api/sheets/category-details?id=cat_1').

Бекенд лізе в потрібну таблицю (або кеш), дістає характеристики конкретної категорії "Вітаміни" і повертає невеликий JSON.

Фронтенд отримує цей JSON і динамічно малює нове вікно, панель або розгортає рядок під категорією (mapper-characteristics.js або mapper-options.js).

Чому це важливо:
З 16 000 категорій користувач за одну сесію фізично взаємодіє максимум з 10-20. Немає сенсу завантажувати і рендерити дані для решти 15 980 категорій.

Для реалізації цього в твоєму мапері потрібно переписати логіку генерації таблиці у mapper-table.js на використання віртуального скролера, а отримання опцій винести в окремий маршрут API, який приймає ID категорії і викликається виключно за кліком.

Ізоляція та Життєвий цикл: Плагін — це завжди окремий файл, який експортує лише одну функцію init(state). Він ніколи не імпортує інші плагіни напряму.

Прибирання за собою (Disposable): Якщо плагін вішає глобальні обробники подій (на window чи document), він обов'язково повертає об'єкт з методом dispose().

Жорсткий формат шапки: Кожен файл починається з правильного шляху та блоку-коментаря з Unicode-рамкою (╔══════╗), де вказана роль файлу (наприклад, 🔌 ПЛАГІН або 🔌 ПЛАГІН (viewless)).

1. Плагін віртуального скролінгу
Цей плагін обманює браузер. Він створює невидиму розпірку (tableSpacer) на всю висоту 16 000 рядків, щоб з'явився повзунок. При події scroll він вираховує математично, які саме 30 рядків зараз на екрані, і генерує HTML тільки для них, зсуваючи контейнер (tableContent) на потрібну висоту через transform: translateY.

JavaScript
// js/pages/mapper/mapper-plugin-virtual-scroll.js
/*
╔════════════════════════════════════════════════════════════════════════╗
║                   ВІРТУАЛЬНИЙ СКРОЛІНГ (MAPPER)                        ║
╠════════════════════════════════════════════════════════════════════════╣
║  Малює лише ті рядки таблиці, які зараз знаходяться в зоні видимості.  ║
║  Запобігає "смерті" браузера при рендері масивів на 10к+ елементів.    ║
║                                                                        ║
║  📋 Вимоги до DOM:                                                     ║
║  ├── state.dom.vScrollWrapper (контейнер зі скролом, relative, fix H)  ║
║  ├── state.dom.vScrollSpacer (порожній div для імітації висоти)        ║
║  └── state.dom.vScrollContent (абсолютний контейнер для <tr>/<div>)    ║
║                                                                        ║
║  🔌 ПЛАГІН                                                             ║
╚════════════════════════════════════════════════════════════════════════╝
*/

export function init(state) {
    const { dom, data } = state;
    const ROW_HEIGHT = 48; // Фіксована висота одного закритого рядка в пікселях
    const BUFFER = 10; // Кількість рядків запасу зверху і знизу для плавного скролу

    function renderVisibleItems() {
        if (!dom.vScrollWrapper || !dom.vScrollContent || !dom.vScrollSpacer) return;
        if (!data.categories || data.categories.length === 0) return;

        const scrollTop = dom.vScrollWrapper.scrollTop;
        const viewportHeight = dom.vScrollWrapper.clientHeight;

        // 1. Математика індексів
        let startIndex = Math.floor(scrollTop / ROW_HEIGHT) - BUFFER;
        startIndex = Math.max(0, startIndex);

        const visibleCount = Math.ceil(viewportHeight / ROW_HEIGHT) + (BUFFER * 2);
        const endIndex = Math.min(data.categories.length, startIndex + visibleCount);

        // 2. Оновлення фізичної висоти розпірки (імітація 16000 рядків)
        // Якщо є розгорнуті рядки, їхню висоту треба рахувати динамічно,
        // але для базового списку беремо фіксовану математику.
        dom.vScrollSpacer.style.height = `${data.categories.length * ROW_HEIGHT}px`;

        // 3. Зсув контейнера з контентом на рівень поточного скролу
        const offsetY = startIndex * ROW_HEIGHT;
        dom.vScrollContent.style.transform = `translateY(${offsetY}px)`;

        // 4. Генерація HTML тільки для видимого "вікна"
        const visibleItems = data.categories.slice(startIndex, endIndex);
        
        // Звертаємось до шини фільтрів: інший плагін або template поверне HTML-рядок
        const html = visibleItems.map(cat => 
            state.applyFilter('onRenderCategoryRow', '', cat)
        ).join('');

        dom.vScrollContent.innerHTML = html;
    }

    // Підписка на фізичний скрол з оптимізацією через requestAnimationFrame
    let ticking = false;
    const onScroll = () => {
        if (!ticking) {
            window.requestAnimationFrame(() => {
                renderVisibleItems();
                ticking = false;
            });
            ticking = true;
        }
    };

    dom.vScrollWrapper?.addEventListener('scroll', onScroll);

    // Реакція на зовнішні події (завантаження бази, фільтрація пошуку)
    state.registerHook('onDidUpdateData', renderVisibleItems, { plugin: 'virtual-scroll' });

    return {
        dispose() {
            dom.vScrollWrapper?.removeEventListener('scroll', onScroll);
        }
    };
}
2. Плагін лінивого завантаження деталей (Lazy Load)
Відповідає виключно за реакцію на клік по категорії. Якщо категорія клікнута вперше — робить API запит до бекенду, зберігає результат в state, міняє прапорець і викликає хук оновлення для віртуального скролера.

JavaScript
// js/pages/mapper/mapper-plugin-lazy-details.js
/*
╔════════════════════════════════════════════════════════════════════════╗
║                   ЛІНИВЕ ЗАВАНТАЖЕННЯ ДЕТАЛЕЙ (MAPPER)                 ║
╠════════════════════════════════════════════════════════════════════════╣
║  Слухає кліки по категоріях. Завантажує характеристики та опції        ║
║  з бекенду лише в момент першого розгортання рядка.                    ║
║                                                                        ║
║  🔌 ПЛАГІН                                                             ║
╚════════════════════════════════════════════════════════════════════════╝
*/

export function init(state) {
    const { dom, data } = state;

    // Делегування: вішаємо один слухач на весь контейнер, а не на 30 рядків
    dom.vScrollContent?.addEventListener('click', async (e) => {
        const btn = e.target.closest('[data-action="toggle-details"]');
        if (!btn) return;

        const row = btn.closest('[data-category-id]');
        if (!row) return;

        const categoryId = row.dataset.categoryId;
        const category = data.categories.find(c => c.id === categoryId);

        if (!category) return;

        // Варіант А: Дані вже є в кеші (state) — просто перемикаємо стан
        if (category._detailsFetched) {
            category._isExpanded = !category._isExpanded;
            state.runHook('onDidUpdateData', null, { plugin: 'lazy-details' });
            return;
        }

        // Варіант Б: Даних немає — тягнемо з сервера
        try {
            // Візуальний стан завантаження на кнопці
            btn.setAttribute('aria-busy', 'true');
            
            // Виклик твого API бекенду
            const response = await fetch(`/api/sheets/category-details?id=${categoryId}`);
            if (!response.ok) throw new Error(`HTTP error! status: ${response.status}`);
            
            const detailsData = await response.json();

            // Записуємо отримані дані в state.data (Єдине джерело правди)
            category.characteristics = detailsData.characteristics;
            category.options = detailsData.options;
            
            // Встановлюємо технічні прапорці
            category._detailsFetched = true;
            category._isExpanded = true;

            // Тригеримо перемальовування
            state.runHook('onDidUpdateData', null, { plugin: 'lazy-details' });

        } catch (error) {
            console.error('[lazy-details] Помилка завантаження:', error);
            // Викликаємо системний тост або алерт
            state.runHook('onError', 'Не вдалося завантажити деталі категорії', { plugin: 'lazy-details' });
        } finally {
            btn.removeAttribute('aria-busy');
        }
    });
}

// js/components/charms/charm-virtual-scroll.js

/*
╔════════════════════════════════════════════════════════════════════════╗
║                       ШАРМ — ВІРТУАЛЬНИЙ СКРОЛ                         ║
╠════════════════════════════════════════════════════════════════════════╣
║  Малює лише видимі елементи списку. Запобігає перевантаженню DOM.      ║
║                                                                        ║
║  📋 Як використовувати:                                                  ║
║  ├── HTML: <div virtual-scroll="48" id="myList"></div>                 ║
║  ├── JS: const el = document.getElementById('myList');                 ║
║  ├── JS: el.vScrollData = [{id: 1, name: 'A'}, ...];                   ║
║  ├── JS: el.vScrollRender = (item) => `<div>${item.name}</div>`;       ║
║  └── JS: el.dispatchEvent(new CustomEvent('vscroll:update'));          ║
║                                                                        ║
║  🔌 ШАРМ                                                               ║
╚════════════════════════════════════════════════════════════════════════╝
*/

export function initVirtualScrollCharm() {
    function initCharm(container) {
        if (container._vScrollInitialized) return;
        container._vScrollInitialized = true;

        const rowHeight = parseInt(container.getAttribute('virtual-scroll') || '40', 10);
        const buffer = 10;

        // Налаштування CSS для контейнера
        container.style.position = 'relative';
        container.style.overflowY = 'auto';

        // Створення розпірки та контенту
        const spacer = document.createElement('div');
        spacer.style.width = '1px';
        spacer.style.opacity = '0';

        const content = document.createElement('div');
        content.style.position = 'absolute';
        content.style.top = '0';
        content.style.left = '0';
        content.style.right = '0';

        container.appendChild(spacer);
        container.appendChild(content);

        // Логіка відмальовування
        const render = () => {
            const data = container.vScrollData || [];
            const renderFn = container.vScrollRender || (() => '');

            if (data.length === 0) {
                spacer.style.height = '0px';
                content.innerHTML = '';
                return;
            }

            spacer.style.height = `${data.length * rowHeight}px`;

            const scrollTop = container.scrollTop;
            const viewportHeight = container.clientHeight || 500;

            let startIndex = Math.max(0, Math.floor(scrollTop / rowHeight) - buffer);
            const visibleCount = Math.ceil(viewportHeight / rowHeight) + (buffer * 2);
            const endIndex = Math.min(data.length, startIndex + visibleCount);

            const offsetY = startIndex * rowHeight;
            content.style.transform = `translateY(${offsetY}px)`;

            const visibleItems = data.slice(startIndex, endIndex);
            content.innerHTML = visibleItems.map(renderFn).join('');
        };

        // Слухачі подій
        container.addEventListener('scroll', () => {
            requestAnimationFrame(render);
        });

        container.addEventListener('vscroll:update', render);
    }

    // 1. Ініціалізуємо існуючі контейнери на сторінці
    document.querySelectorAll('[virtual-scroll]').forEach(initCharm);

    // 2. Слідкуємо за динамічною появою нових контейнерів
    const observer = new MutationObserver((mutations) => {
        mutations.forEach(m => {
            m.addedNodes.forEach(node => {
                if (node.nodeType === 1) {
                    if (node.hasAttribute('virtual-scroll')) initCharm(node);
                    node.querySelectorAll('[virtual-scroll]').forEach(initCharm);
                }
            });
        });
    });
    
    observer.observe(document.body, { childList: true, subtree: true });
}

// 3. Тригериш подію, щоб Шарм зрозумів: "дані є, починай малювати"
listElement.dispatchEvent(new CustomEvent('vscroll:update'));
Що робити з тегом <select> і 24 800 опціями
На стандартному HTML-тезі <select> з тегами <option> це НЕ ПРАЦЮВАТИМЕ.

Причина: <select> — це системний компонент, який відмальовується операційною системою (Windows, macOS), а не чистим рушієм браузера. Туди фізично неможливо вбудувати віртуальний скролінг, зсув через CSS transform чи підміну DOM-вузлів "на льоту". Якщо ти запхаєш 24 800 <option> у нативний селект, сторінка зависне намертво.

Єдине рішення (через архітектуру pinguin-v5):
У твоїй структурі вже є файли js/components/forms/select.js, dropdown.js та css/components/forms/custom-select.css. Ти мусиш використовувати кастомні дропдауни (псевдо-селекти).

Як це працює:

Ти генеруєш не <select>, а звичайну кнопку або input, яка виглядає як селект.

При кліку на неї відкривається абсолютно спозиційований список (div.dropdown-menu).

Саме на цей div.dropdown-menu ти вішаєш атрибут virtual-scroll="40" і згодовуєш йому масив з 24 800 опцій.

Шарм віртуального скролу ідеально підхопить цей div, і в списку, що випав, буде малюватися лише 20 опцій одночасно, підміняючись при прокрутці. Нічого не гальмуватиме.

Варіант А: "Абсолютний Шарм" (Нуль JS на сторінці)

1. Онови файл js/components/charms/charm-virtual-scroll.js:

JavaScript
// Знаходимо шаблон всередині контейнера
const templateEl = container.querySelector('template');
const templateStr = templateEl ? templateEl.innerHTML : '';

// Якщо є атрибут data-source, Шарм сам тягне дані з сервера
const sourceUrl = container.getAttribute('data-source');
if (sourceUrl && templateStr) {
    container.classList.add('is-loading'); // Можна повісити спінер
    
    fetch(sourceUrl)
        .then(res => res.json())
        .then(data => {
            container.vScrollData = data;
            // Проста підстановка змінних з масиву в {шаблон}
            container.vScrollRender = (item) => {
                return templateStr.replace(/\{(\w+)\}/g, (_, key) => item[key] || '');
            };
            container.dispatchEvent(new CustomEvent('vscroll:update'));
            container.classList.remove('is-loading');
        })
        .catch(err => console.error('[VirtualScroll] Помилка:', err));
}
